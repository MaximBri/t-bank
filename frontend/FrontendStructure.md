# Структура фронтенд-части проекта

Фронтенд проекта реализован на `React + TypeScript` и организован по подходу `Feature-Sliced Design (FSD)`. Исходный код разделён на уровни ответственности: приложение, страницы, виджеты, фичи, сущности и общий переиспользуемый слой.

## Общая структура

```text
frontend/
├── src/
│   ├── app/                 # Инициализация приложения и глобальные провайдеры
│   │   ├── App.tsx
│   │   └── providers/
│   │       └── router.tsx
│   │
│   ├── pages/               # Страницы приложения
│   │   ├── home/
│   │   ├── login/
│   │   ├── register/
│   │   ├── event/
│   │   └── not-found/
│   │
│   ├── widgets/             # Крупные составные UI-блоки
│   │   ├── event-list/
│   │   └── event-filters/
│   │
│   ├── features/            # Пользовательские сценарии и бизнес-функции
│   │   ├── sign-in-by-credentials/
│   │   ├── sign-up-by-credentials/
│   │   └── create-event/
│   │
│   ├── entities/            # Базовые бизнес-сущности предметной области
│   │
│   ├── shared/              # Общие переиспользуемые модули
│   │   ├── api/             # Клиент API и запросы
│   │   ├── assets/          # Иконки и статические ресурсы
│   │   ├── config/          # Конфигурация приложения
│   │   ├── lib/             # Вспомогательные утилиты и общая логика
│   │   ├── routes/          # Константы маршрутов
│   │   └── ui/              # Базовые UI-компоненты
│   │
│   ├── main.tsx             # Точка входа
│   ├── styles.css           # Глобальные стили
│   ├── testSetup.ts         # Настройка тестового окружения
│   └── vite-env.d.ts
│
├── package.json
├── vite.config.ts
├── tailwind.config.js
└── tsconfig.json
```

## Назначение основных слоёв

- `app` содержит корневую инициализацию приложения, маршрутизацию и глобальные провайдеры.
- `pages` описывает отдельные экраны приложения: главную страницу, страницу события, вход, регистрацию и страницу 404.
- `widgets` включает крупные визуальные блоки, состоящие из нескольких компонентов и логики, например список событий и фильтры.
- `features` хранит пользовательские сценарии, такие как авторизация, регистрация и создание события.
- `entities` предназначен для описания предметных сущностей. В текущей версии проекта слой подготовлен, но пока используется минимально.
- `shared` содержит переиспользуемую инфраструктуру: UI-компоненты, формы, маршруты, иконки, утилиты и общие настройки.

## Внутренняя структура модулей

Большинство модулей внутри `features` и `widgets` дополнительно разделены на подпапки:

- `ui` — визуальные компоненты
- `model` — типы, схемы, состояния и бизнес-данные
- `lib` — вспомогательная логика и утилиты
- `index.ts` — публичная точка экспорта модуля

Пример:

```text
features/create-event/
├── ui/
├── model/
├── lib/
└── index.ts
```
