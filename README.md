# Название проекта: T-Ивент

## Аннотация проекта:
Разработка веб-приложения для совместного управления бюджетом события или поездки, которое позволяет участникам фиксировать расходы, автоматически рассчитывать взаиморасчёты и прозрачно отслеживать общий бюджет без ручных подсчётов.

## Итоговый продукт:

Результат (продукт)  
**Веб-приложение.**

## Цель - сокращение транзакций между людьми

### Обязательные возможности:
– Регистрация и авторизация пользователей.  
– Создание события / поездки: название, даты, валюта, описание.  
– Роли: «владелец» и «участник».  
– Приглашение участников по ссылке, e-mail, подтверждение участия.  
– Категории бюджета: дефолтные и пользовательские.  
– Добавление расходов: кто платил, за кого, категория, сумма, комментарий.
– Автоматический расчёт долей и отображение «кто кому сколько должен».
– Пересчёт сальдо после добавления или изменения расходов.  
– История событий и действий участников.  
– Экспорт итогов в CSV.

## Список задач, что вы видите:

### Описание функционала, который нужно реализовать:

**Планирование** – владелец создаёт событие / поездку, указывает даты, валюту, категории, приглашает участников.  

**Ведение расходов** – участники добавляют расходы, при необходимости прикладывают чек, система распределяет суммы между участниками и пересчитывает баланс группы.  

**Итоги** – на отдельной вкладке отображаются взаиморасчёты и итоговый баланс: кто кому сколько должен.  

**История** – все действия участников фиксируются в ленте событий.  

### Техническая структура

#### Текущая реализация в репозитории
– **Монорепозиторий** с разделением на `frontend` и `backend`. Отдельная `e2e / qa`-часть пока не выделена.  
– **Бэкенд:** Java 25, Maven, Spring Boot 3.4.4, Spring Web, Spring Data JPA, PostgreSQL драйвер.  
– **Фронтенд:** TypeScript 5.7, React 19, React DOM 19, Vite 6, `@vitejs/plugin-react`.  
– **База данных:** PostgreSQL 16 в `docker-compose.yml`, локальная конфигурация через `application.yml`.  
– **Тесты:** на текущий момент в репозитории подключён только `spring-boot-starter-test` на бэкенде; frontend/e2e-инфраструктура пока не добавлена.  

#### Планируемый стек / целевая архитектура
– **Бэкенд:** Spring Boot, PostgreSQL, миграции схемы БД, OpenAPI.  
– **Фронтенд:** SPA на TypeScript с роутингом, управлением состоянием и HTTP-клиентом.  
– **QA:** Postman-коллекция, e2e smoke / regression сценарии, тест-кейсы и чек-листы.  
– **Безопасность:** аутентификация и авторизация, разграничение доступа к данным события.  
– **Отказоустойчивость:** транзакции БД и защита от конфликтов при пересчёте балансов.  
– **Документация:** инструкция по запуску проекта, описание API, демо-данные, дизайн-макеты.

### Критерии приёмки результата / продукта:
– Пользователь может создать событие / поездку и пригласить участников.  
– Участники могут подтверждать участие и добавлять расходы.  
– После добавления расхода корректно пересчитываются доли и баланс участников.  
– Во вкладке итогов корректно отображается «кто кому сколько должен».  
– История изменений и действий по событию / поездке сохраняется и доступна к просмотру.  
– Для финальной версии должны быть доступны OpenAPI-спека, Postman-коллекция и демо-данные.
