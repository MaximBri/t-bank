# Example Bug Report

## Title
Отсутствует валидация логина при вводе запрещённых символов в форме авторизации

## Severity (Серьёзность дефекта)
- [ ] Высокая
- [ ] Средняя
- [x] Низкая

## Description
В форме авторизации не отображается сообщение об ошибке валидации при вводе запрещённых символов `.` и `\` в поле логина.

## Preconditions
1. Перейти на страницу `http://dev.domain.app/auth`
2. Убедиться, что пользователь не авторизован
3. Открыть форму авторизации

## Steps to Reproduce
1. Ввести в поле логина значение `test.\user`
2. Перевести фокус на другое поле или нажать кнопку авторизации
3. Обратить внимание на состояние поля логина

## Expected Result (Ожидаемый результат)
Под полем логина отображается сообщение об ошибке валидации.

## Actual Result (Фактический результат)
Сообщение об ошибке валидации не отображается.
Поле визуально не показывает наличие ошибки.

## Environment
- Браузер: Chrome `145.0.XXXX.XXX`
- ОС: Linux
- Стенд: `http://dev.domain.app`
