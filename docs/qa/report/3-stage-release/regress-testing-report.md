# Отчет по регрессионному тестированию

## 1. Краткий итог

Проведено повторное ручное регрессионное тестирование основных пользовательских сценариев после исправления блокирующего дефекта регистрации через frontend.

Ранее критичный дефект [`BUG-008`](../../bug-report/bug-008.md) больше не является блокером: регистрация и авторизация через frontend теперь позволяют пройти базовые сценарии.

В ходе повторной проверки выявлены новые дефекты, которые влияют на полноту прохождения регрессионных сценариев:

- [`BUG-011`](../../bug-report/bug-011.md) — после успешного добавления события его изображение не отображается в карточке события;
- [`BUG-012`](../../bug-report/bug-012.md) — редактирование расхода возвращает `500 Internal Server Error`;
- [`BUG-013`](../../bug-report/bug-013.md) — редактирование события возвращает `500 Internal Server Error`;
- [`BUG-014`](../../bug-report/bug-014.md) — фильтрация событий по статусу на главной странице не применяется;
- [`BUG-015`](../../bug-report/bug-015.md) — в форме `textarea`/`multiline input` создания события кнопка очистки текста загораживает элемент скроллбара при длинном значении текста;
- [`BUG-016`](../../bug-report/bug-016.md) — в форме создания события отсутствует frontend-валидация длины поля, из-за чего backend возвращает `500 Internal Server Error`;
- [`BUG-017`](../../bug-report/bug-017.md) — в форме создания события длинное название категории ломает layout чипа и выходит за границы контейнера;
- [`BUG-018`](../../bug-report/bug-018.md) — длинный заголовок расхода в списке расходов ломает layout и перекрывает кнопки управления расходом;
- [`BUG-019`](../../bug-report/bug-019.md) — участник не может присоединиться по пригласительной ссылке;
- [`BUG-020`](../../bug-report/bug-020.md) — длинное название события на странице события ломает layout загаловка и перекрывает кнопки управления;
- [`BUG-021`](../../bug-report/bug-021.md) — в личном кабинете форма редактирования профиля не подставляет актуальные данные и не отправляет запрос на сохранение;
- [`BUG-022`](../../bug-report/bug-022.md) — в личном кабинете форма изменения пароля очищается после отправки формы без запроса на бэкенд;
- [`BUG-023`](../../bug-report/bug-023.md) — на странице события кнопка `завершить событие` не работает.

Основные регрессионные блокеры на текущем этапе:
- невозможность завершить сценарий присоединения второго пользователя по пригласительной ссылки из-за [`BUG-019`](../../bug-report/bug-019.md);
- невозможность проверить редактирование события из-за [`BUG-013`](../../bug-report/bug-013.md);
- невозможность проверить редактирование расхода из-за [`BUG-012`](../../bug-report/bug-012.md);
- невозможность завершить событие через UI из-за [`BUG-023`](../../bug-report/bug-023.md).

Сценарии, зависящие от корректного сценария приглашения и от последующих многопользовательских действий, отмечены как `Blocked` и требуют повторной проверки после исправления [`BUG-019`](../../bug-report/bug-019.md).

## 4. Вывод

После исправления [`BUG-008`](../../bug-report/bug-008.md) сценарии регистрации, авторизации, создания события, добавления расхода и выхода из аккаунта стали доступны для повторной проверки. При этом полный основной пользовательский сценарий продукта по-прежнему не подтвержден. Сценарии профиля выявили отдельные дефекты [`BUG-021`](../../bug-report/bug-021.md) и [`BUG-022`](../../bug-report/bug-022.md), а сценарий завершения события — дефект [`BUG-023`](../../bug-report/bug-023.md).

Регрессионная проверка выполнена, но часть сценариев осталась в статусе `Blocked` из-за открытых дефектов:
- [`BUG-019`](../../bug-report/bug-019.md) блокирует сценария приглашения и все зависящие от него многопользовательских сценарии;
- [`BUG-013`](../../bug-report/bug-013.md) ломает редактирование события;
- [`BUG-012`](../../bug-report/bug-012.md) ломает редактирование расхода;
- [`BUG-023`](../../bug-report/bug-023.md) ломает завершение события владельцем.

Дополнительно в ходе регрессии зафиксированы UI-, validation- и profile-дефекты, которые не всегда блокируют сценарий целиком, но ухудшают качество продукта и требуют исправления: [`BUG-011`](../../bug-report/bug-011.md), [`BUG-014`](../../bug-report/bug-014.md), [`BUG-015`](../../bug-report/bug-015.md), [`BUG-016`](../../bug-report/bug-016.md), [`BUG-017`](../../bug-report/bug-017.md), [`BUG-018`](../../bug-report/bug-018.md), [`BUG-020`](../../bug-report/bug-020.md), [`BUG-021`](../../bug-report/bug-021.md), [`BUG-022`](../../bug-report/bug-022.md).

Полная повторная проверка blocked-сценариев должна быть выполнена после исправления [`BUG-019`](../../bug-report/bug-019.md).
