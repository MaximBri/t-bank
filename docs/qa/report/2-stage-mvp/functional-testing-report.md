# Отчет по функциональному тестированию

## 1. Краткий итог

Проведено ручное функциональное тестирование основных пользовательских сценариев на текущем состоянии MVP.

Основной блокер: регистрация через frontend не проходит из-за CORS-ошибки в браузере / проблемы с миграцией некоторых полей бэкенда.

Связанный дефект: [`BUG-008`](../../bug-report/bug-008.md).

Из-за блокировки регистрации невозможно полноценно проверить сценарии, которые требуют успешной авторизации:
- вход в приложение;
- создание и просмотр событий;
- приглашение участников;
- добавление расходов;
- взаиморасчеты;
- личный кабинет;
- действия с участниками и событиями.

Сценарии, которые требуют авторизованного пользователя, отмечены как `Blocked` и требуют повторной проверки после исправления [`BUG-008`](../../bug-report/bug-008.md).
