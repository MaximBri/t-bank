# Отчет по UI-тестированию

## 1. Краткий итог

Проведено ручное UI-тестирование текущей версии frontend.

На момент проверки пользовательские сценарии были заблокированы: регистрация через frontend не проходила из-за CORS-ошибки в браузере / проблемы с миграцией некоторых полей.

Связанный дефект-блокер: [`BUG-008`](../../bug-report/bug-008.md).

Что удалось проверить для пользовательских сценариев:
- страница логина открывается;
- страница регистрации открывается;
- публичные auth-экраны доступны без авторизации.

Что не удалось полноценно проверить:
- успешную регистрацию через frontend;
- вход после регистрации;
- переход в защищенную часть приложения;
- создание и просмотр событий;
- работу расходов, участников, взаиморасчетов, профиля и других сценариев после авторизации.

Причина: эти сценарии требуют успешной авторизации.

Чтобы не оставлять защищенные экраны совсем без проверки, дополнительно был временно отключен AuthGuard на локальном frontend. После этого защищенные экраны были проверены статически: только открытие страниц и базовое визуальное отображение.

Стоит учесть, что статическая проверка не подтверждает корректность бизнес-сценариев, загрузку данных, отправку форм и backend-интеграцию.

## 4. Вывод

Полная UI-проверка пользовательских сценариев на текущем состоянии MVP заблокирована из-за дефекта авторизации.

Часть экранов удалось проверить только как публичные auth-экраны или статически после временного отключения AuthGuard, без полноценной backend-интеграции.

Повторная полная проверка frontend + backend сценариев должна быть выполнена после исправления [`BUG-008`](../../bug-report/bug-008.md).
