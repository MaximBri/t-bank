# Smoke Checklist

## 1. Цель
Быстро проверить, что критичный пользовательский сценарий работает после нового билда и продукт готов к дальнейшему тестированию/демонстрации/использованию.

## 2. Предусловия
- Доступны frontend и backend.
- Подготовлены минимум 2 тестовых пользователя.
- Доступна рабочая база данных.

## 3. Правила выполнения
- Чеклист может выполняться сверху вниз.
- Проверяется только критичный happy path.
- Для каждой проверки фиксируется один из статусов: Pass / Fail / Blocked.
