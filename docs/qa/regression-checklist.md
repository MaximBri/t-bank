# Regression Checklist

## 1. Цель
Проверить, что после изменений основной функционал приложения работает стабильно, ключевые домены и базовые ограничения доступа не сломаны.

## 2. Предусловия
- Доступны frontend и backend.
- Подготовлены минимум 2 тестовых пользователя.
- Подготовлено тестовое событие для проверки.
- Подготовлены данные для проверки приглашений, расходов и взаиморасчетов.

## 3. Правила выполнения
- Чеклист выполняется по доменам сверху вниз.
- Проверяются основные позитивные сценарии и базовые ограничения доступа.
- Для каждой проверки фиксируется один из статусов: Pass / Fail / Blocked.
