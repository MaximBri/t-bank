## Bug ID
`BUG-015`

## Title
В форме `textarea`/`multiline input` создания события кнопка очистки текста загораживает элемент скроллбара при длинном значении текста

## Severity (Серьёзность дефекта)
- [ ] Высокая
- [x] Средняя
- [ ] Низкая

## Description
В модальном окне `Создание события` и `Добавления расхода` при вводе длинного текста в поле названия события и/или описания кнопка очистки (`крестик`) отображается некорректно и визуально накладываетсся на скроллбар.

Проблема наблюдается:
- в поле `Описание`

Кнопка очистки смещается и выглядит визуально неаккуратно относительно поля ввода, загораживая скроллбар

## Preconditions
1. Frontend запущен локально.
2. Backend запущен локально.
3. Пользователь авторизован.
4. Открыто модальное окно `Создание события`.

## Steps to Reproduce
1. Открыть страницу `Мои события`.
2. Нажать `Создать событие`.
3. Ввести длинное значение в поле `Название события`.
4. Ввести длинное значение в поле `Описание`.
5. Проверить отображение кнопки очистки (`крестика`) в полях.

Также поведение повторяется для формы `Добавления расхода`. 

## Steps to Reproduce
1. Открыть страницу `Мои события`.
2. Нажать `Создать событие`.
3. Перейти к форме `Добавления расхода`.
4. Ввести длинное значение в поле `Комментарий`.
5. Проверить отображение кнопки очистки (`крестика`) в полях.

## Expected Result (Ожидаемый результат)
Кнопка очистки отображается корректно, не наезжает на контент поля и визуально выровнена внутри элемента ввода.

## Actual Result (Фактический результат)
Кнопка очистки отображается некорректно при длинном значении и визуально портит форму.
