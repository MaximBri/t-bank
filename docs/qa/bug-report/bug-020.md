## Bug ID
`BUG-020`

## Title
Длинное название события на странице события ломает layout загаловка и перекрывает кнопки управления

## Severity (Серьёзность дефекта)
- [ ] Высокая
- [ ] Средняя
- [x] Низкая

## Description
На странице события длинное название события отображается без ограничения длины и ломает layout верхнего блока события.

Из-за этого:
- заголовок выходит за пределы нормального layout
- кнопки управления событием (`Редактировать`, `Завершить событие`) смещаются и отображаются некорректно
- страница становится трудной для комфоротного использования

## Preconditions
1. Frontend запущен локально.
2. Backend запущен локально.
3. Пользователь авторизован.

## Steps to Reproduce
1. Создать событие с очень длинным названием, например,`abcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabc
`
2. Открыть страницу события.
3. Проверить отображение заголовка события.
4. Проверить расположение и доступность кнопок управления.

## Expected Result (Ожидаемый результат)
Длинное название события корректно ограничивается, переносится или обрезается без поломки layout-a.
Кнопки управления остаются доступными и корректно выровнены.

## Actual Result (Фактический результат)
Длинное название события ломает layout и затрудняет использование кнопок управления.
