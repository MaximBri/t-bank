## Bug ID
`BUG-018`

## Title
Длинный заголовок расхода в списке расходов ломает layout и перекрывает кнопки управления расходом

## Severity (Серьёзность дефекта)
- [x] Высокая
- [ ] Средняя
- [ ] Низкая

## Description
На странице события в блоке списка расходов длинный заголовок расхода отображается без ограничения длины и ломает layout карточки расхода.

Из-за этого:
- текст выходит за границы нормального layout
- кнопки действий (`редактировать`, `удалить` и другие) становятся частично перекрытыми или вовсе недоступными

## Preconditions
1. Frontend запущен локально.
2. Backend запущен локально.
3. Пользователь авторизован.
4. Существует событие с расходом, у которого очень длинный заголовок.

## Steps to Reproduce
1. Открыть страницу события.
2. Добавить расход с очень длинным заголовком.
3. Сохранить расход.
4. Открыть список расходов.
5. Проверить отображение карточки расхода и доступность кнопок действий.

## Expected Result (Ожидаемый результат)
Длинный заголовок расхода корректно ограничивается, переносится или обрезается без поломки layout.  
Кнопки действий остаются доступными и визуально не перекрываются.

## Actual Result (Фактический результат)
Длинный заголовок расхода ломает layout карточки и мешает пользоваться кнопками действий.

## Environment
- Стенд: local
- Frontend UI
