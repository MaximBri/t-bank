## Bug ID
`BUG-010`

## Title
Для frontend окружения не зафиксированы package manager, его версия и project-level парметры для npm

## Severity (Серьёзность дефекта)
- [ ] Высокая
- [ ] Средняя
- [x] Низкая

## Description
Во frontend-части проекта npm используется фактически, но параметры npm-окружения проекта не зафиксированы явно.

По текущему состоянию репозитория:

- в `frontend/package.json` отсутствует поле `packageManager`
- project-level значение `min-release-age` не задано
- во frontend нет project-level `.npmrc`
- в `frontend/README.md` указано только `npm 10+`
- в `frontend/README.md` для установки используется `npm install`

На фоне актуальных supply-chain атак это является security-риском для локальных- и CI-окружений.

## Preconditions
1. Открыт frontend проекта в ветке `dev` версии коммита `ee5ac33618a5c4a245e16920d0197102ead013e3`.
2. Доступны файлы `package.json`, `package-lock.json`, `README.md`.

## Steps to Reproduce
1. Открыть `frontend/package.json`.
2. Проверить наличие поля `packageManager`.
3. Проверить наличие project-level `.npmrc`.
4. Выполнить команду:

```bash
npm config get min-release-age --location=project
```

5. Открыть `frontend/README.md`.
6. Проверить требования к npm и команду установки зависимостей.
7. Убедиться, что в проекте присутствует `package-lock.json`.

## Expected Result (Ожидаемый результат)
- Параметры npm-окружения проекта зафиксированы и однозначно определены на уровне репозитория / frontend-части.
- Есть возможность установить `min-release-age`

## Actual Result (Фактический результат)
В репозитории явно не зафиксированы:

- package manager
- точная версия package manager
- project-level npm-настройки
- project-level значение `min-release-age`


## Environment
- npm `10+`
- node `v20+`
- Linux local environment
- Стенд: local
