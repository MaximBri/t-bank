## Bug ID
`BUG-014`

## Title
Фильтрация событий по статусу на главной странице не применяется

## Severity (Серьёзность дефекта)
- [x] Высокая
- [ ] Средняя
- [ ] Низкая

## Description
На странице `Мои события` при переключении фильтра по статусу (`Активно`, `Запланировано`, `Завершено`) frontend формирует запросы с корректным query parameter `status`, но список событий не фильтруется по выбранному статусу.

Наблюдается отправка запросов:

- `events?status=active`
- `events?status=planned`
- `events?status=completed`

При этом в списке продолжают отображаться события с другими статусами.

## Preconditions
1. Frontend запущен локально.
2. Backend запущен локально.
3. Пользователь авторизован.
4. У пользователя есть несколько событий с разными статусами.

## Steps to Reproduce
1. Открыть страницу `Мои события`.
2. Выбрать фильтр `Активно`.
3. Проверить список событий.
4. Выбрать фильтр `Запланировано`.
5. Проверить список событий.
6. Выбрать фильтр `Завершено`.
7. Проверить список событий.

## Expected Result (Ожидаемый результат)
При выборе фильтра в списке остаются только события с выбранным статусом.

## Actual Result (Фактический результат)
Фильтр по статусу визуально переключается, запросы с `status` формируются, но список событий не соответствует выбранному статусу.

## Environment
- Стенд: local
- Frontend UI
- Backend API
