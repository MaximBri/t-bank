## Bug ID
`BUG-017`

## Title
В форме создания события длинное название категории ломает layout чипа и выходит за границы контейнера

## Severity (Серьёзность дефекта)
- [ ] Высокая
- [ ] Средняя
- [x] Низкая

## Description
В модальном окне `Создание события` при добавлении категории с очень длинным названием чип категории не ограничивается по длине, не переносится корректно и визуально ломает layout формы.

Проблема состоит из двух частей:
- отсутствует ограничение длины названия категории на frontend
- длинный чип выходит за границы контейнера и перекрывает layout

## Preconditions
1. Frontend запущен локально.
2. Backend запущен локально.
3. Пользователь авторизован.
4. Открыто модальное окно `Создание события`.

## Steps to Reproduce
1. Открыть страницу `Мои события`.
2. Нажать `Создать событие`.
3. В поле добавления категории ввести очень длинное значение, например:

```text
256abcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabca
```

4. Нажать `Добавить`.
5. Проверить отображение чипа категории.

## Expected Result (Ожидаемый результат)
Frontend ограничивает допустимую длину названия категории либо корректно обрабатывает длинное значение без поломки layout-а.

## Actual Result (Фактический результат)
Длинное название категории добавляется без ограничений, чип растягивается и ломает layout формы.
