## Bug ID
`BUG-016`

## Title
В форме создания события отсутствует frontend-валидация длины поля, из-за чего backend возвращает `500 Internal Server Error`

## Severity (Серьёзность дефекта)
- [x] Высокая
- [ ] Средняя
- [ ] Низкая

## Description
В модальном окне `Создание события` отсутствует frontend-валидация максимальной длины значения.  
При вводе слишком длинной строки форма позволяет отправить запрос, после чего backend возвращает:

```json
{"message":"An unexpected error occurred","status":500}
```

Проблема воспроизводится, например, при вводе значения:

```text
256abcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabcabca
```

## Preconditions
1. Frontend запущен локально.
2. Backend запущен локально.
3. Пользователь авторизован.
4. Открыто модальное окно `Создание события`.

## Steps to Reproduce
1. Открыть страницу `Мои события`.
2. Нажать `Создать событие`.
3. Ввести слишком длинное значение в одно из текстовых полей формы.
4. Отправить форму.

## Expected Result (Ожидаемый результат)
Frontend валидирует длину поля до отправки формы и показывает пользователю понятную ошибку валидации.  

## Actual Result (Фактический результат)
Форма отправляется без frontend-валидации, backend возвращает:

```json
{"message":"An unexpected error occurred","status":500}
```

## Environment
- Стенд: local
