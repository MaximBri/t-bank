## Bug ID
`BUG-022`

## Title
В личном кабинете форма изменения пароля очищается после submit-а без запроса на бэкенд

## Severity (Серьёзность дефекта)
- [ ] Высокая
- [x] Средняя
- [ ] Низкая

## Description
На вкладке `Безопасность` в личном кабинете сценарий изменения пароля фактически не работает.

После заполнения полей и нажатия `Изменить пароль`:
- request на backend не отправляется;
- форма очищается;
- пароль пользователя не меняется.

## Preconditions
1. Frontend запущен локально.
2. Backend запущен локально.
3. Пользователь авторизован.

## Steps to Reproduce
1. Открыть страницу `Мой профиль`.
2. Перейти во вкладку `Безопасность`.
3. Заполнить поля:
   - `Текущий пароль`
   - `Новый пароль`
   - `Подтвердите новый пароль`
4. Нажать `Изменить пароль`.
5. Проверить network и состояние формы.

## Expected Result (Ожидаемый результат)
- После submit отправляется request на изменение пароля.
- При успешном выполнении пользователь получает подтверждение, новый пароль работает.

## Actual Result (Фактический результат)
- Request на изменение пароля не отправляется.
- Форма очищается после submit.
- Пароль пользователя не меняется.

## Environment
- Стенд: local
