## Bug ID
`BUG-021`

## Title
В личном кабинете форма редактирования профиля не подставляет актуальные данные и не отправляет запрос на сохранение

## Severity (Серьёзность дефекта)
- [ ] Высокая
- [x] Средняя
- [ ] Низкая

## Description
На странице профиля данные пользователя отображаются некорректно и сценарий обновления профиля фактически не работает.

Наблюдается несколько проблем одновременно:
- в заголовке профиля отображаются две разные почты;
- актуальная почта не подставляется корректно в поле email формы редактирования;
- после ввода новых значений и попытки сохранения request на обновление профиля не отправляется;
- после перезагрузки страницы введенные значения сбрасываются.

## Preconditions
1. Frontend запущен локально.
2. Backend запущен локально.
3. Пользователь авторизован.
4. Открыта страница `Мой профиль`.

## Steps to Reproduce
1. Открыть страницу `Мой профиль` -> вкладка `Профиль`
2. Проверить, какие значения подставлены в поля формы.
3. Нажать `Редактировать профиль`.
4. Изменить данные профиля.
5. Нажать кнопку сохранения.
6. Проверить network и состояние формы.
7. Перезагрузить страницу.

## Expected Result (Ожидаемый результат)
- В форме редактирования профиля подставляются актуальные данные пользователя.
- После сохранения отправляется request на обновление профиля.
- После успешного сохранения новые значения сохраняются и отображаются после перезагрузки страницы.

## Actual Result (Фактический результат)
- Данные в профиле отображаются не в соответствии с данными профиля.
- Поля формы не подставляют корректные значения.
- Request на обновление профиля не отправляется.
- После перезагрузки страницы введенные значения сбрасываются.
