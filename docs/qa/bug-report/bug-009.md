## Bug ID
`BUG-009`

## Title
Кнопки управления событием некорректно отображаются при ширине viewport `1409px` и ниже

## Severity (Серьёзность дефекта)
- [ ] Высокая
- [ ] Средняя
- [x] Низкая

## Description
На странице события в блоке управления событием некорректно отображаются кнопки действий:

- `Покинуть событие`
- `Завершить событие`

При ширине viewport `1409px` и ниже текст кнопки переносится на две строки и визуально выходит за пределы нормального состояния кнопки.


## Preconditions
1. Открывается стенд `https://t-event.front-craft.ru/`
2. Пользователь авторизован
3. У пользователя есть доступ к событию

## Steps to Reproduce
1. Открыть страницу события:
   `https://t-event.front-craft.ru/events/?section=expenses`
2. Установить ширину viewport `1409px` или меньше в devTools
3. Обратить внимание на карточку события в верхней части страницы
4. Проверить отображение кнопок:
   - `Завершить событие`
   - `Покинуть событие`

## Expected Result (Ожидаемый результат)
Кнопки управления событием отображаются корректно:
- текст находится внутри кнопки;
- текст читается без визуальных наложений;
- кнопки имеют консистентное выравнивание;
- если есть перенос текста, то не ломает внешний вид элемента.

## Actual Result (Фактический результат)
При ширине viewport `1409px` и ниже кнопки управления событием отображаются некорректно.

Кнопка `Покинуть событие`:
- текст переносится на две строки.

Кнопка `Завершить событие`:
- текст переносится на две строки;

## Environment
- Браузер (Chromium-based browser, chromimum version `146.0.7680.211`, Linux/Pop!_OS 24.04 LTS x86_64)
- Стенд: деплой (`https://t-event.front-craft.ru/`)
- Страница: `https://t-event.front-craft.ru/events/?section=expenses`
- Viewport: `1409 × 800` и ниже по ширине
