# Test Case

## Test Case ID
`TC-015`

## Title
Редактирование события владельцем

## Feature / Module
Events / `PATCH /events/{eventId}`

## Preconditions
1. Пользователь `qa_owner_01` авторизован.
2. Пользователь `qa_owner_01` является владельцем события.
3. Для редактирования события используются данные: новое название `QA Встреча обновлена`, новое описание `Обновленное описание события`, валидная дата начала и валидная дата окончания.
4. Пользователь находится на главной странице приложения.

## Steps
1. Открыть нужное событие на главной странице.
2. На странице события нажать кнопку Редактировать.
3. Дождаться открытия модального окна редактирования события.
4. Изменить название события.
5. Изменить описание события.
6. Обновить даты события.
7. Нажать кнопку Сохранить.

## Expected Result (Ожидаемый результат)
Данные события успешно обновлены.
Отображается уведомление об успешном обновлении события.
На странице события отображаются новые данные.
Ошибки валидации и 5XX отсутствуют.

## Postconditions
- Для выбранного события сохранены обновленные данные.
