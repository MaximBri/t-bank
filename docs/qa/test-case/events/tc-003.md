# Test Case

## Test Case ID
`TC-003`

## Title
Создание нового события

## Feature / Module
Events / `POST /events`

## Preconditions
1. Пользователь `qa_owner_01` авторизован.
2. Для создания события используются данные: title `QA Встреча в кафе`, description `QA Описание 123!`, валидная будущая дата начала и дата окончания позже даты начала.

## Steps
1. Открыть экран создания события.
2. Заполнить обязательные поля валидными значениями.
3. Заполнить поле Название.
4. Заполнить поле Дата начала.
5. Заполнить поле Дата конца.
6. Сохранить событие.

## Expected Result (Ожидаемый результат)
Событие успешно создано с уведомлением в верху экрана на главной странице.
Событие доступно в списке событий пользователя.
Ошибки валидации и 5XX отсутствуют.
