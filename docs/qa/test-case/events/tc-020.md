# Test Case

## Test Case ID
`TC-020`

## Title
Завершение события владельцем

## Feature / Module
Events / `POST|PATCH finish event` / UI button `Завершить событие`

## Preconditions
1. Пользователь `qa_owner_01` авторизован.
2. Пользователь `qa_owner_01` является владельцем события.
3. Событие находится в статусе `Активно` или `Запланировано`.
4. На странице события отображается кнопка `Завершить событие`.

## Steps
1. Открыть нужное событие на главной странице.
2. Убедиться, что событие доступно владельцу для управления.
3. Нажать кнопку `Завершить событие`.
4. Проверить network и состояние страницы после нажатия.
5. Если отображается подтверждение, подтвердить завершение события.

## Expected Result (Ожидаемый результат)
После нажатия кнопки отправляется request на завершение события.
Статус события изменяется на `Завершено`.
На странице и в списке событий отображается обновленный статус события.
Кнопка завершения больше не отображается как доступная для повторного действия.
Ошибки 5XX отсутствуют.

## Postconditions
- Выбранное событие переведено в статус `Завершено`.
