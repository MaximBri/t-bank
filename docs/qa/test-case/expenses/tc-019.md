# Test Case

## Test Case ID
`TC-019`

## Title
Оспаривание расхода

## Feature / Module
Expenses / `PATCH /expenses/{expenseId}/status`

## Preconditions
1. Пользователь `qa_member_01` авторизован.
2. Пользователь `qa_member_01` является участником события.
3. В событии есть расход, доступный для оспаривания (имеется иконка ! напротив расхода)

## Steps
1. Открыть нужное событие на главной странице.
2. Перейти в раздел Расходы.
3. Найти расход, доступный для оспаривания.
4. Нажать кнопку оспаривания расхода.
5. Дождаться открытия модального окна Оспаривание расхода.
6. Заполнить поле Причина оспаривания.
7. Нажать кнопку Оспорить.

## Expected Result (Ожидаемый результат)
Открывается модальное окно Оспаривание расхода.
После отправки причины расход переводится в статус На проверке.
Отображается уведомление о переводе расхода на проверку.
Обновленный статус расхода отображается в списке расходов.
Под оспоренным расходом отображается причина оспаривания и время создания оспаривания.
Ошибки 5XX отсутствуют.

## Postconditions
- У выбранного расхода сохранен статус На проверке.
