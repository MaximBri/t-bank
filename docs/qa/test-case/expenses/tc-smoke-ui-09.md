# UI Test Case

## Test Case ID
`TC-SMOKE-UI-09`

## Title
Открытие формы добавления расхода

## Feature / Module
UI / Expenses / Create Expense Form

## Preconditions
1. Frontend запущен и доступен
2. Пользователь авторизован в приложении.
3. Существует доступное пользователю событие.

## Steps
1. Перейти на страницу `/events/{eventId}?section=expenses`, где `{eventId}` это идентификатор существующего события.
2. Нажать кнопку открытия формы добавления расхода.
3. Проверить наличие основных полей формы.

## Expected Result (Ожидаемый результат)
Форма добавления расхода открывается и отображается корректно и согласно макету.
