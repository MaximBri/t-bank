# Test Case

## Test Case ID
`TC-007`

## Title
Добавление нового расхода в событие

## Feature / Module
Expenses / `POST /events/{eventId}/expenses`

## Preconditions
1. Пользователь `qa_owner_01` авторизован.
2. В событии есть минимум 2 участника.
3. Для создания расхода используются данные: title `Ужин`, amount `2000`, валидная категория.

## Steps
1. Открыть раздел расходов события.
2. Нажать добавление нового расхода.
3. Заполнить название, сумму, категорию, подтверждающий документ и участников для разделения.
4. Добавить расход.

## Expected Result (Ожидаемый результат)
Новый расход успешно создан и отображается в списке расходов события.
Oшибки валидации и 5XX отсутствуют.

## Postconditions
- В событии создан новый расход `Ужин`.
