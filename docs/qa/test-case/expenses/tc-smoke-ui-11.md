# UI Test Case

## Test Case ID
`TC-SMOKE-UI-11`

## Title
Открытие формы оспаривания расхода

## Feature / Module
UI / Expenses / Dispute Expense Form

## Preconditions
1. Frontend запущен и доступен
2. Пользователь авторизован в приложении.
3. Существует доступное пользователю событие.
4. В событии существует расход, доступный для оспаривания.

## Steps
1. Перейти на страницу `/events/{eventId}?section=expenses`, где `{eventId}` это идентификатор существующего события.
2. Выбрать расход, доступный для оспаривания.
3. Открыть форму оспаривания расхода.
4. Проверить наличие основных полей формы.

## Expected Result (Ожидаемый результат)
Форма оспаривания расхода открывается и отображается корректно и согласно макету.
