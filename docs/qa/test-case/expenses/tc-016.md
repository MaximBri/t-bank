# Test Case

## Test Case ID
`TC-016`

## Title
Редактирование расхода владельцем, если расход на проверке

## Feature / Module
Expenses / Редактирование расхода на проверке

## Preconditions
1. Пользователь `qa_owner_01` авторизован.
2. Пользователь `qa_owner_01` является владельцем события.
3. В событии есть расход в статусе На проверке.
4. Для исправления расхода используются данные: корректная сумма, валидная категория, новый подтверждающий документ и комментарий.
5. Пользователь находится на главной странице приложения.

## Steps
1. Открыть нужное событие на главной странице.
2. Убедиться, что открыта вкладка Расходы.
3. Найти расход в статусе На проверке.
4. Нажать кнопку редактирования расхода.
5. Дождаться открытия модального окна исправления расхода.
6. Изменить сумму расхода.
7. При необходимости изменить категорию.
8. Заменить подтверждающий документ.
9. Заполнить комментарий.
10. Нажать кнопку Отправить.

## Expected Result (Ожидаемый результат)
Данные расхода успешно обновлены.
Отображается уведомление об успешном обновлении расхода.
Обновленный расход отображается в списке расходов события.
Ошибки валидации и 5XX отсутствуют.

## Postconditions
- Для выбранного расхода сохранены обновленные данные.
