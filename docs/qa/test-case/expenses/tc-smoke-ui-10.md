# UI Test Case

## Test Case ID
`TC-SMOKE-UI-10`

## Title
Открытие формы исправления расхода

## Feature / Module
UI / Expenses / Edit Expense Form

## Preconditions
1. Frontend запущен и доступен
2. Пользователь авторизован в приложении.
3. Существует доступное пользователю событие.
4. В событии существует доступный для редактирования расход.

## Steps
1. Перейти на страницу `/events/{eventId}?section=expenses`, где `{eventId}` это идентификатор существующего события.
2. Выбрать расход, доступный для редактирования.
3. Открыть форму исправления расхода.
4. Проверить наличие основных полей формы.

## Expected Result (Ожидаемый результат)
Форма исправления расхода открывается и отображается корректно и согласно макету.
