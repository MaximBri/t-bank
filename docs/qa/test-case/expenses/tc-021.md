# Test Case

## Test Case ID
`TC-021`

## Title
Проверка выбора категории при добавлении расхода

## Feature / Module
Expenses / Category selection in expense form

## Preconditions
1. Пользователь `qa_owner_01` авторизован.
2. У пользователя есть созданное событие.
3. Открыта форма `Добавление расхода` в событии.

## Steps
1. Открыть событие пользователя.
2. Открыть форму `Добавление расхода`.
3. Открыть выпадающий список `Категория`.
4. Проверить, что в списке отображаются доступные категории расходов.
5. Выбрать одну из категорий.
6. Заполнить остальные обязательные поля формы валидными значениями.
7. Сохранить расход.
8. Проверить, что расход создан с выбранной категорией.

## Expected Result (Ожидаемый результат)
Поле `Категория` отображает доступные категории расходов корректно.
Пользователь может выбрать категорию при создании расхода.
После сохранения расход отображается с выбранной категорией.
Ошибки валидации, layout-дефекты и 5XX отсутствуют.
