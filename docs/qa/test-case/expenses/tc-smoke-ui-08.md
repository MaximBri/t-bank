# UI Test Case

## Test Case ID
`TC-SMOKE-UI-08`

## Title
Открытие раздела события "Расходы"

## Feature / Module
UI / Events / Expenses Section

## Preconditions
1. Frontend запущен и доступен
2. Пользователь авторизован в приложении.
3. Существует доступное пользователю событие.

## Steps
1. Перейти на страницу `/events/{eventId}?section=expenses`, где `{eventId}` это идентификатор существующего события.
2. Дождаться полной загрузки страницы.
3. Проверить, что раздел "Расходы" открыт и отображается корректно.

## Expected Result (Ожидаемый результат)
Раздел события "Расходы" открывается и отображается корректно и согласно макету.
