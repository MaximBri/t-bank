# Test Case

## Test Case ID
`TC-018`

## Title
Запрет выхода из события при наличии незакрытых задолженностей

## Feature / Module
Participants / `GET /events/{eventId}/leave/check`

## Preconditions
1. Пользователь `qa_member_01` авторизован.
2. Пользователь `qa_member_01` является участником события.
3. У пользователя `qa_member_01` есть хотя бы один незакрытый долг или расход, который мешает выходу из события.

## Steps
1. Открыть нужное событие на главной странице.
2. Нажать кнопку Покинуть событие.
3. Дождаться открытия модального окна с запретом выхода.
4. Нажать кнопку К взаиморасчетам.

## Expected Result (Ожидаемый результат)
Открывается модальное окно с сообщением о невозможности выхода из события.
Пользователю отображается причина запрета выхода.
После нажатия кнопки К взаиморасчетам открывается раздел взаиморасчетов.
Пользователь остается участником события.
Ошибки 5XX отсутствуют.

## Postconditions
- Пользователь `qa_member_01` остается участником выбранного события.
