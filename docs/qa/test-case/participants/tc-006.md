# Test Case

## Test Case ID
`TC-006`

## Title
Проверка списка участников события

## Feature / Module
Participants / `GET /events/{eventId}/participants`

## Preconditions
1. Пользователь `qa_owner_01` авторизован.
2. В событии есть минимум 2 участника: владелец и второй пользователь.

## Steps
1. Открыть карточку события.
2. Перейти в раздел участников на страцнице ивента.

## Expected Result (Ожидаемый результат)
В списке участников отображаются владелец события и второй пользователь.
Статусы участников отображаются корректно.
Ошибки 5XX отсутствуют.
