# Test Case

## Test Case ID
`TC-008`

## Title
Проверка взаиморасчетов после добавления расхода

## Feature / Module
Settlements / `GET /events/{eventId}/settlements`

## Preconditions
1. В событии есть минимум один подтвержденный расход, разделенный между двумя участниками.
2. Пользователь `qa_owner_01` авторизован.

## Steps
1. Открыть карточку события.
2. Перейти в раздел взаиморасчетов.

## Expected Result (Ожидаемый результат)
В разделе взаиморасчетов отображается актуальный долг между участниками.
Сумма расчета соответствует добавленному расходу и логике деления.
Ошибки 5XX отсутствуют.
