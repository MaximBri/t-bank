# Test Case

## Test Case ID
`TC-009`

## Title
Отметка долга как оплаченного

## Feature / Module
Settlements / `POST /settlements/{settlementId}/pay`

## Preconditions
1. В событии есть активный расчет между участниками.
2. Авторизован пользователь, который должен выполнить оплату.
3. Используется валидный `settlementId` активного расчета.

## Steps
1. Открыть раздел взаиморасчетов.
2. Найти активный долг пользователя.
3. Выполнить действие отметки долга как оплаченного.

## Expected Result (Ожидаемый результат)
Статус расчета успешно изменен на состояние ожидания подтверждения. Ошибки 5XX отсутствуют.
