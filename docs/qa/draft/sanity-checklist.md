# Sanity Checklist

## 1. Цель
Быстро проверить, что конкретный фикс или доработка работает корректно, а затронутый домен и ближайшие связанные действия не сломаны.

## 2. Предусловия
- Доступны frontend и backend на целевом окружении.
- Известен фикс, доработка или дефект, который нужно проверить.
- Определен затронутый домен.
- Подготовлены нужные тестовые данные и тестовый пользователь.

## 3. Правила выполнения
- Чеклист заполняется под конкретное изменение.
- Проверяются измененный домен и 1-2 ближайших связанных действия.
- Нерелевантные пункты удаляются или помечаются как N/A.
- Для каждой проверки фиксируется один из статусов: Pass / Fail / Blocked.
- Если в затронутой зоне появляется блокер или 5XX, sanity считается неуспешным.

## 4. Таблица проверок

| № | Проверка | Статус | Комментарий |
|---|---|---|---|
| SAN-01 | Authentication: авторизация под валидным пользователем |  |  |
| SAN-02 | {Измененный домен}: открытие нужного экрана или раздела |  |  |
| SAN-03 | {Измененный домен}: выполнение основного действия по фиксу |  |  |
| SAN-04 | {Измененный домен}: проверка сохранения или отображения результата |  |  |
| SAN-05 | {Связанный домен 1}: проверка ближайшего связанного сценария |  |  |
| SAN-06 | {Связанный домен 2}: проверка второго связанного сценария |  |  |
| SAN-07 | {Измененный домен}: повторное выполнение действия или повторное открытие результата |  |  |
| SAN-08 | {Измененный домен}: проверка отсутствия блокирующих ошибок и 5XX |  |  |

## 5. Критерий прохождения sanity
- Фикс или доработка работают корректно.
- В затронутом домене нет Blocked.
- В затронутой зоне нет 5XX.
